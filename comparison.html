Detection Logic (DFS)

In our Framework, deadlock detection uses a Depth First Search (DFS) on the Wait-For Graph.

Graph Construction: Create nodes for Processes and Edges for dependencies.
Wait Edge: Process A ➔ Resource X (Held by B) implies Edge A ➔ B.
Cycle Check: If DFS traces a path Back to Start (e.g., A ➔ B ➔ C ➔ A), a Deadlock exists.

Prevention vs Avoidance

Prevention: Structurally creates a system where one of the 4 Coffman conditions cannot hold (e.g., Hierarchy breaks Circular Wait).

Avoidance: Dynamically checks states (like Chandy-Misra or Banker's Algorithm) to steer clear of unsafe states during execution.
